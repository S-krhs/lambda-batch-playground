# Libs

app/domain 非依存の汎用処理を置く領域です。依存の重さで `utils`(純粋処理。軽量な npm 依存のみ)と `browser`(Playwright-core など browser 実行依存)に分けます。

- `gacha`、`string`、`date` など処理の関心でディレクトリを切る。browser 実行のような重い依存が必要な処理は `utils` に混ぜず、責務単位の package へ分ける。
- 1 ファイルに型、検証、アルゴリズム、公開 API をまとめない。責務が複数ある場合は関心ごとにファイルを分ける。
- ファイル先頭コメントには、そのファイル固有の責務だけを書く。モジュール全体の説明を各ファイルに流用しない。
- integration や app の都合が混ざり始めた場合は境界を見直す。DI は複雑さを減らす具体的な理由がある場合だけ使う。
- 乱数を主目的にする lib は、抽選ロジック本体を `random: () => number` を受け取れる形にして検証可能にし、`Math.random` は公開 API の境界でだけ既定値として閉じ込める。
